### 1. 论文的核心问题与整体含义
*   **研究动机**：现有的多模态视觉语言模型（VLM）在处理医学图像时面临两大挑战：一是**视觉证据极度稀疏**，即图像中只有极小区域对临床决策有用，其余部分多为冗余；二是**计算开销与干扰**，密集的视觉 Token 不仅增加了推理负担，还可能引入干扰模式，降低推理准确性。
*   **核心问题**：如何建立一个统一的强化学习（RL）框架，既能主动识别关键视觉区域（Token 剪枝），又能基于精简后的 Token 进行高质量的医学多模态推理？

### 2. 方法论：ViToS 框架
论文提出了一种名为 **ViToS** 的双流强化学习框架，其核心思想是将视觉 Token 剪枝（VTP）建模为策略引导的证据选择过程。
*   **双流架构**：
    *   **定位分支（Localization Branch）**：策略模型首先生成思维链（CoT）并预测关键区域的边界框（Bounding Box）。
    *   **Token 稀疏推理分支**：根据预测的边界框进行“感知定位的视觉 Token 剪枝（GTP）”。保留框内 Token，并将框外背景 Token 基于语义相似度融合到框内 Token 中，从而在减少 Token 数量的同时保留全局上下文。
*   **关键技术：交叉反馈顺序优化（Cross-feedback Sequential Optimization）**：
    *   为了解决双流任务中的梯度冲突和耦合学习问题，论文引入了交叉反馈机制。
    *   **奖励设计**：包含格式奖励、空间定位奖励（IoU）和任务准确率奖励。
    *   **优化策略**：先优化定位分支，利用推理分支的准确率作为反馈信号；待定位稳定后，再冻结定位策略，优化稀疏 Token 下的推理能力。
*   **算法基础**：采用 **GRPO**（Group Relative Policy Optimization）算法进行大规模强化学习训练。

### 3. 实验设计

*   **对比方法**：

### 6. 主要结论与发现
*   **性能超越全量 Token**：ViToS 在将视觉 Token 减少到原始长度的 **77%** 的情况下，性能反而超过了全量 Token 模型。在 Lingshu-7B 上达到了 108.27% 的相对性能。
*   **推理加速**：由于 Token 数量减少且兼容 vLLM 推理引擎，ViToS 在 7B 模型上实现了约 **4.5 倍** 的推理加速，在 32B 模型上实现了约 **3.4 倍** 加速。
*   **冗余度正相关**：实验发现，视觉冗余度越高的模态（如眼底照相、X光），通过 Token 剪枝获得的性能提升越明显。

### 7. 优点与亮点
*   **端到端优化**：不同于传统的启发式剪枝，ViToS 将剪枝过程纳入 RL 闭环，使模型学会“为了推理而剪枝”。
*   **Token 融合机制**：通过语义相似度将背景信息融合而非直接丢弃，有效缓解了因定位不准导致的性能波动。
*   **训练效率高**：通过顺序优化和冻结视觉编码器预计算 Token 的策略，显著降低了 RL 训练的资源消耗。

### 8. 不足与局限
*   **分辨率限制**：目前的实验基于标准分辨率医学图像。对于极高分辨率（如超大病理切片 WSI）图像，定位动作空间会急剧增大，策略学习可能面临收敛困难。
*   **定位依赖性**：虽然有 Token 融合机制缓冲，但推理分支仍高度依赖定位分支的准确性，若定位完全偏离，模型性能仍受限。
*   **泛化至非医疗领域**：该方法针对医学图像的“稀疏证据”特性设计，在视觉信息密集的通用场景（如自然风景、复杂街道）中的有效性尚未验证。
