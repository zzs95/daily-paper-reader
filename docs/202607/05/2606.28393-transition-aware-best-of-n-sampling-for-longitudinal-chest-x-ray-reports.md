### 1. 论文的核心问题与整体含义（研究动机和背景）
在临床放射学中，胸部 X 光（CXR）的解读通常不是孤立的。放射科医生会对比患者之前的检查结果，重点描述病情的变化（如“病灶吸收”、“新发阴影”或“维持稳定”）。然而，现有的视觉语言模型（VLM）在生成报告时，大多将单张图像视为独立个体，或者仅在生成阶段引入历史信息。

本论文的核心问题是：**如何在不重新训练模型的情况下，在推理阶段通过采样策略（Best-of-N）让生成的报告更符合临床上的“纵向演变逻辑”？** 论文提出了一种“状态转移感知”（Transition-Aware）的筛选机制，旨在从多个候选报告中选出那个最能体现从“过去”到“现在”合理变化的报告。

### 2. 论文提出的方法论
该方法的核心思想是将报告的变化建模为向量空间中的“位移”，并利用训练集中的真实变化规律作为参考。

*   **报告的集合表示**：将报告拆分为句子，利用预训练的 Sentence Transformer（`all-mpnet-base-v2`）将每个句子映射为 $d$ 维向量。一份报告被表示为一个无序的句子嵌入集合。
*   **定向集合距离（Directional Set Distance）**：定义了四种函数 $T(A, B)$，用于计算从先验报告集合 $A$ 到当前报告集合 $B$ 的定向向量：
    1.  **Mean-Shift**：两个集合质心的差值。
    2.  **Novelty**：当前报告中每个句子相对于先验报告中最近邻居的平均残差。
    3.  **Dir-Hausdorff**：捕捉集合间差异最大的点对之间的位移。
    4.  **Cost-OT**：基于代价加权的最优传输（Optimal Transport）位移。
*   **状态转移库（Transition Bank）**：在推理前，预先计算并缓存训练集中所有真实（先验 $\to$ 当前）报告对的转移向量。
*   **采样与筛选流程**：
    1.  对于测试图像，生成 $N$ 个候选报告。
    2.  计算每个候选报告相对于患者真实先验报告的“候选转移向量”。
    3.  计算该向量与“状态转移库”中所有真实向量的余弦距离。
    4.  通过 `min`（寻找最相似的一个）或 `kNN`（寻找最相似的 $k$ 个的均值）进行聚合评分。
    5.  选择得分最高（距离最小）的候选报告作为最终输出。

### 3. 实验设计
*   **数据集**：使用 **ReXGradient-160K** 数据集，筛选出具有至少两次连续检查的患者，构建了包含 22,745 个训练转移对和 1,727 个测试转移对的纵向队列。
*   **生成模型（基座）**：测试了三种主流模型：Gemini-2.5-Flash-Lite、Gemini-3.1-Flash-Lite-preview 和 Mistral-Small-2603。
*   **提示词策略（Prompts）**：
    *   **P1**：仅图像（零样本）。
    *   **P2**：少样本（提供 5 个随机训练示例）。

*   **Benchmark 与对比方法**：以 **Random Selection**（从 $N=5$ 个候选者中随机抽取）作为基准线。

*   **候选池大小限制**：实验中 $N$ 仅为 5，在更大的 $N$（如 50 或 100）下性能提升可能会更显著，但也会增加推理成本。
