### 1. 核心问题与整体含义（研究动机和背景）
在 LLM 的后训练（Post-training）阶段，针对特定领域（如数学、代码、指令遵循）的强化学习（RL）能显著提升模型性能。然而，**如何将这些分散在不同专家模型中的能力集成到一个单一模型中**，仍是一个悬而未决的难题。
*   **现有方法的缺陷**：
    *   **Mix-RL（混合强化学习）**：多领域信号干扰，产生“跷跷板效应”，性能低于单领域专家。
    *   **Cascade RL（级联强化学习）**：顺序训练导致灾难性遗忘，且训练周期长、不稳定。
    *   **Off-Policy Finetune（离线微调）**：存在严重的“曝光偏差”（Exposure Bias）。
    *   **Param-Merge（参数合并）**：在权重空间融合，往往导致模型不稳定，难以发挥专家全效。

### 2. 论文提出的方法论：MOPD
MOPD（多教师在线策略蒸馏）的核心思想是在**策略空间（Policy Space）**而非权重空间进行能力集成。

*   **三阶段流程**：
    1.  **通用 SFT**：获得基础能力模型。
    2.  **领域专家训练**：并行训练多个特定领域的 RL 专家（教师）。
    3.  **MOPD 集成**：学生模型从 SFT 检查点初始化，在自己的生成分布（On-policy）上运行，根据 Prompt 路由到对应的教师模型，通过最小化**反向 KL 散度**吸收教师的 token 级概率分布。
*   **关键技术细节**：
    *   **密集监督信号**：不同于传统 RL 的轨迹级奖励（Reward），MOPD 使用 Token 级的对数概率差作为优势函数（Advantage），大幅降低方差。
    *   **两种实现方式**：
        *   **Policy-Gradient（策略梯度）**：将教师与学生的对数概率差作为 Advantage 注入 PPO/GRPO 框架。
        *   **Top-k 蒸馏**：仅对教师概率最高的 k 个 token 进行蒸馏，减少通信开销并保持精度。
    *   **同源教师（Same-origin Teachers）**：强调教师和学生必须从同一个 SFT 模型演化而来，以确保分布接近，维持训练稳定性。

*   **能力继承率高**：MOPD 能够继承各专家教师 91%-95% 的能力，且各领域表现均衡，无明显短板。

*   **稳定性**：同源教师提供的低 KL 环境是训练不崩溃的关键。

### 7. 优点
*   **架构解耦**：允许各领域团队并行开发、独立调优，互不干扰，极大地提升了工业界大模型的研发效率。
*   **消除偏差**：On-policy 设计彻底解决了离线蒸馏中的曝光偏差问题。
*   **工程友好**：通过异步服务化教师模型，解决了多模型并存时的显存和计算延迟瓶颈。

### 8. 不足与局限
*   **路由依赖**：MOPD 依赖于准确的 Prompt 路由（将任务分配给正确的教师），对于边界模糊的混合任务，路由错误可能导致错误的引导信号。
*   **存储开销**：虽然训练是并行的，但需要同时维护和部署多个专家教师模型作为服务，对推理服务器的存储和管理有一定要求。
*   **多样性风险**：过度蒸馏可能导致学生模型的策略过于向教师靠拢，虽然性能提升，但可能降低模型在开放域生成的创造性或多样性（尽管文中显示熵值相对稳定）。
