### 1. 论文的核心问题与整体含义
论文针对生成式语言模型（LMs）中的**“词表瓶颈”（Vocabulary Bottleneck）**问题展开研究。传统模型使用线性增长的嵌入矩阵来表示 token，这导致参数量随词表规模剧增，限制了模型在多语言扩展和罕见词处理上的效率。
虽然此前已有研究在“仅编码器”模型中使用哈希技术（如 HashFormer），但由于存在**多对一冲突（Collisions）**，导致解码器无法从共享的哈希索引中唯一确定生成的 token，因此无法用于因果生成任务。本论文提出了 **MultiHashFormer**，旨在通过多重哈希签名机制，在保持极低参数占用的同时，实现高性能的自回归文本生成。

### 2. 论文提出的方法论
MultiHashFormer 的核心思想是将每个 token 表示为一个唯一的**多 ID 哈希签名（Multi-ID Hash Signature）**，并设计了配套的编解码架构：

*   **多重哈希索引（Multi-Hash Indexing）**：利用 $H$ 个独立的哈希函数（如 MurmurHash3）将每个 token 映射为一组离散 ID 序列。通过增加哈希函数的数量，理论上可以支持极大规模的虚拟词表（如 4 个函数、每桶 1.6 万个位置可支持 65 兆个唯一签名），从而彻底消除冲突。
*   **哈希编码器（Hash Encoder）**：采用**门控组合嵌入（Gated Compositional Embedding）**。它将 $H$ 个哈希坐标对应的嵌入向量通过一个轻量级的瓶颈网络（Bottleneck Network）进行加权融合，生成一个统一的潜向量输入给 Transformer 骨干网络。
*   **哈希解码器（Hash Decoder）**：引入**级联预测器（Cascaded Predictor）**。它不再一次性预测 token，而是自回归地逐个预测哈希签名中的 ID。通过递归的“级联混合器”（Cascade Mixer），前一个预测的哈希 ID 会作为上下文信息传递给下一个 ID 的预测。
*   **概率建模与推理**：训练时优化各坐标独立概率的乘积；推理时，通过在有效词表（Vactl）内进行对数概率累加和归一化，确保生成的签名始终对应真实的 token。
