### 1. 核心问题与研究动机
*   **核心问题**：视觉语言模型（VLM）在进行有监督微调（SFT）以提升感知能力（如OCR、图像描述）时，往往会导致推理能力（如数学、科学推理）显著下降。这种现象被称为**“推理税”（Reasoning Tax）**。
*   **研究动机**：作者假设预训练模型在不同深度形成了特征层级（浅层负责感知，深层负责抽象推理）。统一的微调破坏了这种层级访问机制。研究旨在探索如何通过恢复和优化跨层级（跨深度）的信息获取，在不牺牲感知增益的前提下挽救推理能力。

### 2. 核心方法论：IADA (Input-Adaptive Depth Aggregation)
*   **核心思想**：在Transformer块之间插入轻量级的聚合模块，根据当前输入动态地从之前的层级中检索并整合特征。
*   **关键技术细节**：
    *   **固定深度聚合（诊断基准）**：首先通过一个可学习的固定查询向量（Query），利用交叉注意力机制从之前所有块的边界提取特征，证明了“恢复跨深度访问”本身就能大幅提升推理。
    *   **输入自适应查询生成**：IADA 改进了固定查询，通过对当前隐藏状态进行均值池化，并经过一个**低秩瓶颈（Low-rank Bottleneck）**映射，生成与输入内容相关的查询向量。
    *   **模态感知路由**：针对视觉和文本标记（Tokens）采用独立的路由流，使用自模态条件（Self-modal conditioning）来决定如何检索历史深度信息。
    *   **低秩门控注入**：检索到的残差信息通过一个初始值为零的门控机制（Sigmoid激活）注入当前层，确保训练初期的稳定性。
    *   **结构互补**：IADA 专注于“跨层路由”，而 LoRA 专注于“层内变换”，两者结合形成了结构上的互补。

### 3. 实验设计
*   **基材模型**：Qwen3-VL-2B-Instruct。
*   **数据集**：使用 LLaVA-Instruct-Mix（25.9万条样本）进行微调。
*   **Benchmark（11个基准测试）**：
    *   **推理类**：MMMU, MathVista, ScienceQA, MMStar, AI2D。
    *   **感知类**：MME, POPE, RealWorldQA, ChartQA, TextVQA, OCRBench。
*   **对比方法**：Pre-trained（不微调）、LoRA-only（标准微调）、AttnRes+LoRA（固定查询聚合）。

### 4. 资源与算力
*   **硬件**：使用了 **1 台 NVIDIA H200 GPU**。
*   **训练配置**：训练 1 个 Epoch，共 16,198 个优化步数，有效 Batch Size 为 16。
*   **精度与技术**：采用 bf16 混合精度训练，并开启了梯度检查点（Gradient Checkpointing）。
*   **参数量**：IADA 模块极其轻量，仅引入约 **0.14 M** 额外参数（相比之下，LoRA 约为 7.6M）。

### 5. 实验数量与充分性
*   **实验规模**：论文进行了主实验对比、消融实验（针对查询设计、模态条件、瓶颈秩、路由粒度等）以及与 LoRA 秩的交互实验。
*   **充分性评价**：实验设计较为**充分且客观**。作者不仅验证了最终方法的有效性，还通过诊断性实验（固定聚合）验证了假设的合理性。消融实验覆盖了架构设计的多个维度（如 Token-level vs Seq-level），增强了结论的说服力。

### 6. 主要结论与发现
*   **推理税确实存在**：仅使用 LoRA 微调使推理均分下降了 5.1 分。
*   **跨深度访问是关键**：简单的固定深度聚合就能恢复大部分丢失的推理能力。
*   **IADA 表现卓越**：在 Qwen3-VL-2B 上，IADA 将推理均分提升了 9.5 分，感知均分提升了 3.3 分（相比 LoRA-only）。
*   **低秩高效性**：秩为 16 的瓶颈效果优于全秩（Full-rank），说明跨深度路由是一个低维决策问题。
*   **能力互补**：IADA 在低秩 LoRA 设置下增益最明显，说明它弥补了参数受限微调中的结构缺陷。

### 7. 优点与亮点
*   **视角新颖**：将推理能力下降归结为“跨深度表示访问受阻”，而非简单的灾难性遗忘。
*   **极其轻量**：仅需 0.14M 参数（约占总可训练参数的 1.8%）即可获得显著收益。
*   **即插即用**：作为 PEFT 的补充，不改变原有的 Transformer 内部计算逻辑。
*   **输入自适应**：通过动态路由解决了固定聚合无法处理不同任务需求的问题。

### 8. 不足与局限
*   **模型泛化性待验证**：实验仅在 Qwen3-VL-2B 单一模型上完成，尚未在更大规模（如 7B, 72B）或其他架构（如 InternVL）上验证。
*   **模态掩码依赖**：依赖于固定的模态掩码（Modality Mask），在视觉与文本特征高度融合的极深层可能不再适用。
*   **评估维度限制**：主要关注标准 Benchmark 分数，未评估对长文本生成质量或自由对话交互的影响。
*   **数据分布单一**：仅使用了一个指令微调数据集，不同数据分布下的表现尚不明确。
