### 1. 核心问题与整体含义（研究动机和背景）
随着生成式 AI 的演进，多模态伪造内容（图像、视频、文本）在社交媒体泛滥。现有的视觉-语言取证方法（如 OmniVL-Guard）大多遵循**封闭世界范式**，即假设模型仅靠自身参数知识即可完成验证。
然而，自包含的多模态大模型（MLLMs）面临三大瓶颈：
*   **知识滞后**：无法处理需要实时外部线索的事件验证（如 2025 年后的新闻）。
*   **分辨率限制**：难以观察局部细微的伪造痕迹（如边缘不连续、纹理异常）。
*   **推理深度不足**：在复杂取证中容易出现“答案正确但推理过程扭曲”的伪成功现象。
**核心目标**：将取证模型从“闭环预测器”转变为“工具增强型智能体”，通过调用外部工具（搜索、缩放、分割等）实现证据驱动的开放世界取证。

### 2. 方法论：核心思想与关键技术
OmniVL-Guard Pro 的核心架构由**工具环境**、**轨迹生成框架**和**强化学习算法**三部分组成：

*   **工具增强环境**：集成了实时事件搜索、局部裁剪/缩放、边缘异常筛选、人脸检测、视频帧提取及基于 SAM3 的像素级分割工具。
*   **树状结构自进化工具轨迹生成（Tree-Structured Self-Evolving Trajectory Generation）**：
    *   **种子引导**：利用专家模型（如 GPT-5.4）作为“引导者”，通过树状搜索和分支剪枝生成高质量初始轨迹。
    *   **自进化扩展**：模型在无引导环境下自主探索，通过质量过滤不断扩充训练集。
    *   **弱提示硬样本合成**：针对复杂样本，由引导者提供模糊提示（如大致区域），再由“重写者”消除提示痕迹，确保模型学习自主推理而非依赖提示。
*   **检查员引导的智能体强化学习（CGARL）**：
    *   **核心公式**：在标准奖励（答案正确性 $R_{ans}$）基础上，引入**过程监督奖励**。
    *   **Checker 机制**：训练一个专门的判别模型（Checker），评估推理轨迹、工具调用与证据链是否逻辑一致。
    *   **奖励函数**：$R = s_{proc} \times (R_{ans} + R_{fmt} + R_{tool\_min}) + R_{rep}$。其中 $s_{proc}$ 是 Checker 给出的过程分，旨在惩罚“瞎猜对”的行为，鼓励高效、一致的推理。
