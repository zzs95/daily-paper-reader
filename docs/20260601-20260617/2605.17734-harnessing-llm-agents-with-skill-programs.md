### 1. 核心问题与整体含义（研究动机和背景）
论文针对当前大语言模型（LLM）智能体在处理复杂、长程任务时的局限性提出了解决方案。
*   **核心问题**：现有的智能体技能（Skills）通常以**纯文本（Textual Guidance）**形式存在。这种方式存在两个弊端：一是“建议性”而非“强制性”，模型在推理时常忽略这些建议；二是缺乏明确的触发机制，模型不知道何时该调用何种技能。
*   **研究动机**：将技能从“被动的文本提示”升级为“**主动的可执行程序函数（Program Functions, PFs）**”，为智能体构建一个外部控制层（Harness），从而实现对智能体决策过程的显式干预和修正。

### 2. 核心方法论
HASP 框架的核心思想是将技能定义为包含触发和干预逻辑的 Python 函数。

*   **关键技术细节：**
    *   **程序函数 (PF) 结构**：每个 PF 包含两个核心方法：`should_activate`（根据当前状态和候选动作判断是否触发）和 `intervene`（执行干预，如修改动作 `MODIFY_ACTION` 或注入上下文 `INJECT_CONTEXT`）。
    *   **智能体控制层 (Harness)**：在推理时，HASP 作为一个外部包装器，拦截智能体提出的原始动作，检索并运行相关的 PFs，最后将修正后的动作或补充信息反馈给智能体。
    *   **三种应用模式**：
        1.  **推理时干预**：无需训练，直接通过 PFs 修正智能体行为。
        2.  **训练后内化**：利用 PF 修正后的高质量轨迹，通过监督微调（SFT）、拒绝采样（RS）或在线策略蒸馏（OPD）将技能“内化”到模型参数中。
        3.  **自我进化**：分析失败案例，自动总结重复的失败模式并生成新的 PFs，经过验证和教师评审后加入技能库。

### 6. 主要结论与发现
*   **性能大幅提升**：在 Web 搜索任务上，仅靠推理时 PF 干预就比 ReAct 智能体提升了 25%；结合训练和进化后，比 Search-R1 提升了 30.4%。
*   **技能内化有效**：模型可以通过训练学习 PF 的修正逻辑。分析显示，一些行为修正类技能（如“减少过度搜索”）在训练后不再需要 PF 触发，说明模型已掌握该技能。
*   **过滤是进化的关键**：在自我进化过程中，如果没有“执行验证”和“教师评审”的严格过滤，技能库会迅速被噪声污染，导致性能大幅下降。

### 7. 优点（亮点）
*   **显式控制**：将模糊的文本技能转化为确定的程序逻辑，解决了智能体“不听指挥”的问题。
*   **高度模块化**：HASP 既可以作为插件用于推理，也可以作为数据生成器用于训练，具有极强的灵活性。
*   **闭环自增**：建立了一套从“失败分析 -> 技能生成 -> 自动验证 -> 评审入库”的完整闭环，使智能体具备持续学习能力。

### 8. 不足与局限
*   **教师模型依赖**：在技能生成的评审阶段和部分复杂的 PF 逻辑中，仍然需要高性能教师模型（如 GPT-4o）的支持。
*   **策略发现能力**：PFs 目前更多是帮助模型“更好地利用已知策略”，而在“发现全新解题策略”方面的贡献相对有限（如在极难的 AIME 数学题上提升较小）。
