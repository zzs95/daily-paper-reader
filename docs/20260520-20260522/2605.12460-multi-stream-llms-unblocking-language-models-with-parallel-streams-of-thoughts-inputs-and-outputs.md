*   **核心瓶颈**：在单流模式下，模型是“阻塞”的——它不能在阅读时行动，不能在写作时对新信息做出反应，也不能在思考时同步读取或操作。
*   **研究动机**：为了打破这种串行限制，作者提出将 LLM 从顺序消息处理转向**多流并行计算**。这使得模型在单次前向传播中能同时处理多个输入流（如用户输入、系统指令、工具反馈）并生成多个输出流（如思考过程、用户回复、工具调用）。

### 2. 论文提出的方法论
核心思想是**多流并行生成（Multi-Stream Parallel Generation）**，将不同角色分配到独立的令牌流中，并保持跨流的因果依赖。

*   **关键技术细节**：
    *   **跨流因果掩码**：在注意力机制中，每个流的令牌可以关注自身流的过去令牌，以及其他所有流在当前时间步之前的令牌。
    *   **流感知位置编码**：采用旋转位置嵌入（RoPE），每个流独立计数，并引入可学习的“流嵌入”（Stream Embedding）来区分不同流的身份。
    *   **交错打包（Interleaved Packing）**：为了提高训练效率，将多个流的令牌按位置交错排列，使注意力矩阵呈现接近下三角的结构，从而兼容 FlashAttention 等加速算子。
    *   **数据构建流水线**：
        1.  **Wait-k 策略**：通过提示工程让模型在用户输入未完成时就开始响应。
        2.  **流表格生成**：直接生成多列同步的表格数据。
        3.  **因果验证与过滤**：使用强模型作为裁判，确保输出流不包含“未来”输入流的信息。
