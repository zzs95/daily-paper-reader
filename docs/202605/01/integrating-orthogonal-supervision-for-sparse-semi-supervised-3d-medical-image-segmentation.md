### 2. 方法论：集成正交监督（IOS）与三解码器框架
该论文提出了一种名为 **IOS (Integrating Orthogonal Supervision)** 的框架，核心思想是**“单一编码器集成学习 + 三解码器解耦流”**：
*   **集成正交监督 (IOS)**：不同于以往为每个视角建立独立模型的方法，IOS 使用一个共享编码器同时从轴向、冠状和矢状三个正交平面的稀疏标注中学习。这种方式能让模型在单一特征空间内整合互补的解剖信息，构建更全面的体积表示。
*   **三解码器框架 (Tri-Decoder Framework)**：
    *   **解码器 $D_\phi$ 与 $D_\xi$**：负责处理“稀疏标注体积”。两者通过**交叉解码器监督（Cross-decoder supervision）**互相学习，利用彼此的预测结果作为伪标签，来挖掘这些体积中未标注切片的语义信息。
    *   **解码器 $D_\theta$**：专门负责处理“完全未标注体积”。它由 $D_\phi$ 和 $D_\xi$ 共同生成的协同伪标签进行引导。
*   **解耦数据流 (Disentangled Data Flow)**：将稀疏标注数据和完全未标注数据的训练流分开，有效缓解了未标注数据中的噪声对标注数据特征提取的负面干扰。
