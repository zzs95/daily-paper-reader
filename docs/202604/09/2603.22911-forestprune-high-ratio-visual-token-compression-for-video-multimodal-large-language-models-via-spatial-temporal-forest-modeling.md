### 1. 核心问题与研究动机
*   **背景**：随着视频多模态大模型（Video MLLMs）的发展，为了追求更强的理解能力，模型通常需要输入数十甚至数百个视频帧，导致视觉 Token 数量激增，带来了巨大的计算开销、内存占用和推理延迟。
*   **核心问题**：现有的 Token 压缩方法（如剪枝或合并）多针对图像设计，在处理视频时往往只关注单帧内的重要性，忽略了视频在时间维度上的连续性和冗余性。
*   **观察**：在高压缩比下，现有方法保留的 Token 在相邻帧之间存在高度相似和冗余，导致性能大幅下降。

### 2. 方法论：时空森林建模 (ForestPrune)
ForestPrune 是一种**无需训练（Training-free）**的 Token 剪枝方法，其核心思想是将视频 Token 建模为“时空森林”。
*   **核心思想**：通过语义相似度、空间位置约束和时间顺序，将跨帧的相似 Token 连接成“树”。树的深度和节点角色（根节点、干节点、叶节点）反映了该 Token 在全局视频序列中的重要性。
*   **关键流程**：
    1.  **候选节点选择**：首先从每帧中提取代表性 Token 作为树节点。
    2.  **时空森林构建**：
        *   计算节点间的余弦相似度（语义约束）。
        *   计算节点间的欧氏距离（空间约束，确保只有位置相近的相似物体被连接）。
        *   遵循时间顺序（时间约束，仅允许后一帧节点连接到前一帧）。
        *   通过阈值过滤建立连接矩阵，形成由多棵树组成的森林。
    3.  **全局 Token 剪枝**：
        *   根据树的深度对树进行排序（深树通常代表视频中持续出现的关键目标）。
        *   优先剪掉“叶子节点”和“尾部节点”，保留“根节点”和“骨干节点”。
        *   这种策略确保了在极高压缩比下，视频的关键动态信息仍能被全局最优地保留。

*   **超参数敏感性**：虽然消融实验显示在一定范围内鲁棒，但语义和空间阈值（$\tau_s, \tau_p$）可能仍需要针对不同风格的视频（如快速运动 vs. 静态监控）进行微调。
