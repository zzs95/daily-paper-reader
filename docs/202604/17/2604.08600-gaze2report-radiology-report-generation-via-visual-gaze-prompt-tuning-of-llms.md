### 1. 核心问题与整体含义（研究动机和背景）
放射科报告生成（RRG）旨在自动将医学影像转化为详细的文字描述。现有方法多采用“图像编码-文本解码”架构，虽然在自然语言生成（NLG）指标上表现尚可，但存在两个核心痛点：
*   **缺乏医学先验：** 纯视觉模型往往忽略了放射科医生在诊断时的临床逻辑和关注重点，导致生成的报告与疾病表现之间的对齐度较差。
*   **注视数据获取难：** 尽管放射科医生的“眼动注视（Eye Gaze）”能提供关键的诊断线索，但在推理阶段（实际临床应用中）获取实时的眼动数据成本极高且不切实际。
**核心目标：** 提出一个能融合医生注视先验、且在推理阶段无需真实眼动输入的报告生成框架。

### 2. 方法论：核心思想与关键技术
论文提出了 **Gaze2Report** 框架，其核心思想是将视觉特征与预测的注视轨迹融合为“视觉-注视提示（Visual-Gaze Prompt）”，并对大语言模型（LLM）进行微调。

*   **视觉令牌生成：** 使用 Vision Transformer (ViT) 将胸部 X 光片切分为非重叠网格，提取空间特征向量。
*   **注视令牌生成（MedGaze）：** 引入辅助的扫描路径预测模型（MedGaze），模拟放射科医生的注视点序列。通过时间聚合（Duration Aggregation）计算每个图像块的注视时长，生成注视嵌入。
*   **图神经网络（GNN）融合：** 构建以图像块为节点的图，节点特征包含视觉、注视和位置编码。利用 GNN 的多层图卷积处理节点间的空间和逻辑关系，最后通过均值池化和线性投影，将图特征映射到 LLM 的特征空间。
*   **多模态提示微调：** 将“生成指令”、“视觉-注视令牌”和“报告令牌”拼接成统一提示符，利用 **LoRA（低秩自适应）** 技术对 Llama2-7B 进行微调。
*   **推理优化：** 在推理阶段，使用 MedGaze 预测的扫描路径代替真实注视数据，解决了数据缺失问题。

### 3. 实验设计

### 6. 主要结论与发现
*   **注视信息的价值：** 引入注视轨迹显著提升了报告的质量，Gaze2Report 在所有主要 NLG 指标上均优于现有 SOTA 模型。
*   **GNN 的优越性：** 相比于简单的特征拼接或交叉注意力，GNN 能更好地捕捉图像块与医生关注点之间的复杂交互。
*   **推理可行性：** 即使在推理时没有真实的眼动仪数据，通过 MedGaze 预测的注视路径依然能显著提升模型性能（优于纯视觉模型）。
*   **临床准确性：** 定性分析显示，该模型生成的报告在描述“胸腔积液”、“主动脉钙化”等具体病灶时更加精准。

### 7. 优点（亮点）
*   **专家经验集成：** 巧妙地将放射科医生的认知过程（注视路径）转化为模型可理解的提示符。
*   **解决实际痛点：** 通过辅助预测模块解决了眼动数据在临床部署中难以获取的“脱节”问题。
*   **高效微调：** 结合了 LLM 的强大生成能力与 LoRA 的高效微调，保证了报告的语言流畅度和专业性。

### 8. 不足与局限
*   **计算开销：** 尽管使用了 LoRA，但引入 GNN 和 LLM（7B 参数）对于某些实时性要求极高的临床场景可能仍显沉重。
