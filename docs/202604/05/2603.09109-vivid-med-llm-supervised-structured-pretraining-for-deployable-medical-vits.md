### 1. 核心问题与研究背景
*   **核心问题**：现有的医学图像预训练方法主要依赖“独热标签（One-hot labels）”或“自由文本（Free-form text）”进行监督。
    *   **独热标签**：忽略了临床发现之间复杂的语义关联（如胸腔积液与肺水肿的共存性）。
    *   **自由文本**：表述差异大、噪声多，难以提取稳定的临床特征。
*   **研究动机**：利用大语言模型（LLM）内置的结构化临床知识作为“教师”，指导视觉 Transformer（ViT）学习。目标是训练出一个高性能、轻量化且在推理时无需依赖重型 LLM 的可部署视觉骨干网络。

### 2. 方法论
VIVID-Med 框架由三个核心组件组成：可训练的 ViT 编码器、结构化预测分解（SPD）投影器和冻结的 LLM 教师。
*   **统一医学模式（UMS）监督**：
    *   将原始临床报告转换为可验证的 **JSON 字段-状态对**（如 `{"Lung Opacity": {"state": "present"}}`）。
    *   **可回答性感知掩码（Answerability-aware Masking）**：通过 JSON 中的布尔值过滤掉无法评估的发现，防止无效梯度干扰优化。
    *   **字段查询训练**：在训练中随机采样临床字段，并对长尾（低频）病症进行过采样（概率 0.6），以增强模型对罕见病的识别能力。
*   **结构化预测分解（SPD）**：
    *   采用类似 Q-Former 的机制，将交叉注意力分解为 $G$ 个正交正则化的查询组。
    *   **正交正则化（Orthogonality Regularization）**：通过损失函数 $L_{ortho}$ 强制不同查询组关注图像中互补的解剖区域（如一组关注肺部，另一组关注心脏）。
*   **训练与部署流程**：
    *   **训练期**：ViT 提取特征，SPD 将其投影至 LLM 空间，LLM 通过预测 UMS-JSON 序列提供监督信号。
    *   **部署期**：完全舍弃 SPD 和 LLM，仅保留轻量级的 ViT 骨干网络，用于下游任务。

*   **实验规模**：涵盖了 4 个主流医学数据集，进行了线性探测（Linear Probing）、零样本迁移和跨模态泛化测试。

### 7. 优点

*   **特征解耦**：SPD 机制通过正交约束强制模型学习互补的视觉特征，避免了特征冗余。

*   **部署友好**：解耦了训练时的复杂性和推理时的简洁性。
