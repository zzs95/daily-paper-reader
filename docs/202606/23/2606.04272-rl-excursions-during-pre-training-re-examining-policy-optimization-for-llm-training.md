### 1. 核心问题与整体含义
本文挑战了当前大语言模型（LLM）训练的“标准范式”——即先进行大规模预训练（Pre-training），再进行有监督微调（SFT），最后才进行强化学习（RL）。作者提出了两个核心疑问：
*   **时机问题**：RL 是否必须在预训练完全结束后才能进行？在预训练的中早期引入 RL 是否有效？
*   **本质作用**：RL 究竟是像近期一些研究认为的那样仅仅在“收敛/锐化”模型已有的分布，还是能够真正扩展模型的分布并挖掘新能力？
研究背景在于，随着推理模型（如 DeepSeek-R1, OpenAI o1）的兴起，理解 RL 在模型能力构建中的角色变得至关重要。

### 2. 方法论：核心思想与算法流程
论文的核心思想是**“预训练中的 RL 偏移（Excursions）”**，即在预训练的各个中间检查点（Checkpoints）直接应用不同的后训练策略。
*   **核心对比路径**：
    1.  **直接 RL (Direct RL)**：从预训练中间点直接进行 RL，不经过 SFT。
    2.  **标准流水线 (SFT → RL)**：遵循传统的先 SFT 后 RL 路径。
    3.  **并行平均更新 (Parallel Averaging)**：作者提出的一种新算法。在每一个训练步中，同时计算 RL 梯度和 SFT 梯度，两者保持独立的 Adam 优化器状态（一阶和二阶矩），然后取两者的更新量平均值来更新模型参数。
*   **关键技术细节**：RL 采用基于可验证奖励的强化学习（RLVR），具体算法为 **GRPO**。奖励函数主要基于数学问题的最终答案正确性（二元奖励）和格式奖励。

### 3. 实验设计
*   **模型与数据**：从零训练了一个 1B 参数的模型（基于 OLMo2 架构），并在 50B 标记的 DOLMino 高质量数据集上进行预训练。
*   **任务场景**：主要集中在数学推理（GSM8K 和 MATH 数据集）。
*   **对比基准 (Benchmarks)**：
    *   **推理能力**：GSM8K, MATH (评估指标为 Pass@1, Pass@8, Pass@32)。
    *   **通用能力**：LAMBADA, HellaSwag, ARC, PIQA, OpenBookQA（用于监测 RL 是否导致通用能力退化）。
*   **对比方法**：包括 Base 模型、仅 SFT（单解）、SFT-Gold（多解）、SFT → RL 以及提出的 Parallel Averaging。

*   **硬件**：使用了 NVIDIA A100 和 H100 GPU。
*   **时长**：预训练耗时数天；GRPO 强化学习训练耗时数天；SFT 耗时数小时。

实验设计非常严谨且充分：
*   **多阶段采样**：在预训练的 4B, 10B, 20B, 44B 等多个时间点提取检查点进行实验。

*   **客观性**：所有 RL 和 SFT 任务均训练至收敛，并考虑了随机种子对早期检查点训练稳定性的影响，确保了对比的公平性。

### 6. 主要结论与发现
*   **RL 介入极早**：在预训练仅 4B tokens 时（远未达到 Chinchilla 最优），直接应用 RL 就能显著提升推理能力，效果甚至能比肩完整的 SFT → RL 流程。
*   **分布扩展 vs. 锐化**：研究发现，RL 本身具有**扩展**模型分布的能力（提升 Pass@32）。此前研究观察到的“锐化效应”（Pass@1 升但 Pass@32 降）实际上是 SFT 阶段造成的，而非 RL 本身的缺陷。
*   **通用能力保护**：直接对 Base 模型做 RL 几乎不会损害其通用语言能力，而 SFT 则会导致明显的通用能力退化。
*   **数据胜过规模**：在处理难题（如 MATH）时，增加预训练中的相关数据比单纯增加模型参数规模更能提升 RL 的效率。
*   **并行更新最优**：提出的 Parallel Averaging 算法在保持通用能力的同时，取得了最优的 Pass@32 表现。

### 7. 优点与亮点
*   **视角独特**：打破了“RL 必须在 SFT 之后”的思维定式，为“强化学习预训练”提供了实证支持。
*   **算法创新**：Parallel Averaging 提供了一种简单且有效地结合 NTP（下文预测）和 RL 目标的方法。
*   **诊断性发现**：发现 Base 模型的 Pass@k 表现可以作为 RL 是否会产生收益的轻量级预测指标。

*   **领域局限**：实验主要集中在数学推理这一具有“可验证奖励”的领域，在代码或通用对话等奖励模糊的领域是否适用尚待验证。
